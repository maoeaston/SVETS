# 会话启动：恢复现场（SVETS 项目专用）

当前阶段只允许读取文件和检查 Git 状态。
不允许修改文件、不允许运行构建或测试、不允许恢复 stash。

---

## 步骤一：按顺序读取

1. `AGENTS.md` — 架构原则、禁止清单、Vibe Coding 开发流程
2. `.continue-here.md` — 上次会话交接的当前状态和下一步

如果 `.continue-here.md` 提到了额外必读文件，再补读。
不要主动扩读大型历史文档（PRD / schema），除非 `.continue-here.md` 明确要求。

---

## 步骤二：检查 Git 现场

```bash
git stash list
git status --short --branch
git log --oneline -5
```

规则：
- 只列出 stash，不要自动 pop
- 当前只在功能分支上工作（feat/xxx），不在 main 上直接开发
- 看到 main 分支上有未提交改动，先停止并说明

---

## 步骤三：输出启动确认摘要

固定用以下格式，不要加额外说明：

```
当前任务：
当前 Vibe Coding 步骤：/vibe-feature / /vibe-impl / 逐步实现第N步 / /vibe-accept
当前分支：
下一步唯一原子动作：
相关文件：
是否存在 stash：
最近提交：
Repo 状态：clean / dirty / stashed
```

**Vibe Coding 步骤说明（判断依据）：**
- 若 `doc/features/<feature>-prd.md` 不存在 → 应先运行 `/vibe-feature`
- 若 PRD 存在但 `doc/features/<feature>-impl.md` 不存在 → 应先运行 `/vibe-impl`
- 若实现文档存在 → 看 `.continue-here.md` 记录的当前步骤编号
- 若所有步骤完成但未验收 → 应运行 `/vibe-accept`

---

## 步骤四：遇到以下情况先停止并说明

- `.continue-here.md` 不存在（说明是全新会话，告知用户）
- `.continue-here.md` 与当前 Git 状态明显冲突
- 存在无法判断用途的 stash
- 工作区有与当前任务直接冲突的未提交改动
- 当前在 main 分支且有未提交改动

完成以上检查后停止，等待用户下一步指令。不要提前开始干活。
