# Key Decisions
- **事件写入事务边界**：DB 部分经 better-sqlite3 singleton connection 自动加入调用者事务；jsonl 不在事务内，由 reducer 幂等性 + 冷启动重放兜底（Step 5 核心约束，详见 assessment-impl.md「事件写入事务边界」专节）
- **assessment-reducer 方案 A**：独立 `assessment-reducer.ts`，幂等（每 apply* 先 SELECT 判已应用），writeEvent 后调，可复用于冷启动回放
- **reducer 是投影唯一写入者**：handler（Step 6b+）只调 writeEvent + applyAssessmentEvent，不另 INSERT assessment_session / assessment_session_question（impl.md Step 6b 的 INSERT 描述已由 reducer 承接）
- **SESSION_STARTED → status=ACTIVE**：无独立 SESSION_ACTIVATED 事件，INIT→ACTIVE 的非 EVENT UPDATE 会让冷启动重放无法重建 ACTIVE，故 reducer 直达 ACTIVE（impl.md「或 ACTIVE」flex note）
- **幂等策略**：INSERT 类用 PK 存在性判（session_id/answer_id/result_id）；UPDATE 类用 last_status_event_id == event.event_id 判（单调重放下充分）
- **测试 seed 用 uuid 派生业务键**：避免 vitest 并行 + sql.js 下 UNIQUE 冲突（详见 memory `feedback_test_seed_unique_keys.md`）
- **测试 drop trg_assessment_session_no_delete**：reducer 单测不验证该触发器，beforeAll drop 后 beforeEach 可清 assessment_session；其他测试文件各自 createTestDb 加载完整 schema 不受影响
